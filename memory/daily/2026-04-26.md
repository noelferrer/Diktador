## Done (later same day) — Framework decision

Brainstormed and locked the framework + STT pipeline. Filed as [[decisions/framework-choice]].

- **Pivoted off Tauri.** New stack: Swift 5.10+ / SwiftUI, single Xcode project, menu bar app (`LSUIElement`), macOS 14.0+ minimum.
- **STT**: WhisperKit (Argmax, MIT, Apple Silicon Core ML / Neural Engine) as default; Groq HTTPS as user-selectable alternative; Groq key stored in Keychain.
- **Hotkey**: soffes/HotKey package. Default proposal: Right-Option held for push-to-talk.
- **Text injection**: hybrid — clipboard-paste primary, CGEvent keystroke fallback.
- **Persistence**: UserDefaults + Keychain for the Groq key.
- **Modules** (per AGENTS.md six rules): recorder / transcriber / hotkey / output / settings / ui.

## Reasoning notes (for future me)

- Why Swift over Tauri: framework speed advantage in dictation is mostly invisible (STT dominates the latency budget at 200–600 ms vs framework overhead at 50–100 ms), but Swift wins on memory footprint, injection reliability in Electron-based target apps, and **access to WhisperKit's Apple Silicon optimization**. The Windows port being deferred (not foreclosed) was the swing factor.
- Why WhisperKit over whisper.cpp sidecar: the entire reason to choose Swift is to access Apple-Silicon-native inference. Picking whisper.cpp sidecar would forfeit that and leave no compelling reason vs Tauri.
- Why Groq stays in the picture: free tier covers personal use, fast for those who add a key, but local default ensures the "no cost ever" floor.
