## Things that bit / Open questions

- **Xcode required for any Phase B onward**, not just Command Line Tools. User had to install ~15 GB Xcode mid-flow. Worth a note in `memory/general.md` so future-me doesn't try to scaffold a Swift app on a CLT-only machine.
- **SwiftPM identity collisions are real and silent** when local-package-dir name lowercases to the same identity as a transitive dep URL's last path component. The fix (rename the consuming directory) is not obvious from the SwiftPM error message — it just silently fails to fetch.
- **APFS case-insensitivity** can collide `Hotkey.swiftmodule` with `HotKey.swiftmodule` and overwrite our build output without warning. Same family of issue.
- **`HotKey.Key` qualifier ambiguity** — `HotKey` exports both a class and an enum at the top level; `HotKey.Key` resolves to a non-existent member of the class. Workaround: `@_exported import enum HotKey.Key`.
- **Workspace hook blocks pushes to main** even with verbal user approval. One-time manual `git push -u origin main` from the user is the path; future feat→main PRs flow normally.
- **`/go` skill needs a "no code changes since last commit?" branch** — it correctly halts on a clean tree, but on a fresh-no-history repo we hit a chicken-and-egg with the no-push-to-main rule. The skill text covers this in its Guardrails section and the user-action workaround worked.
